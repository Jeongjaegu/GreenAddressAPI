Simple JS login example¶

Here we demonstrate how you can implement a simple JavaScript web application which logs in to GreenAddress wallet using your mnemonic, and checks balance.

Requirements¶

To implement this example, we use the following libraries, so make sure you download them first:
  • Autobahn|JS 0.8.2
  • bitcoinjs-lib 0.2.0
  • weilu’s BIP39 JS implementation, v1.0.2
  • jQuery 2.1.1

Overview¶

To log in to a GreenAddress wallet, you need a private key derived from a mnemonic. The address corresponding to the key is passed to get_challenge() call to retrieve a ‘challenge’ string to be signed by the private key, which is then verified by the authenticate() call.

This example implements a simple GreenAddress login with mnemonic as the input, using the get_challenge() and authenticate() calls.

Compiling bitcoinjs-lib¶

To compile bitcoinjs-lib into a browser-usable form, you need to have node.js installed. Compiling it with npm is rather simple, you just need to execute the following inside the bitcoinjs-lib-0.2.0 directory:
  1. npm install
  2. npm run-script compile

This will create the bitcoinjs-min.js file for you, which you can use in your application.

Compiling weilu’s bip39¶

As above, you need to use node.js to compile the bip39 implementation, which is a bit more involved because this package doesn’t include a pre-made script for compiling into a browser-usable form. Run the following inside the bip39-1.0.2 directory:
  1. npm install
  2. npm install browserify
  3. ./node_modules/.bin/browserify -s BIP39 ./index.js > bip39.js

This will create the bip39.js file.

Deriving the private key from mnemonic¶

Because GreenAddress uses BIP39 for deriving keys from the mnemonic, you can use any BIP39 implementation to do this. We’ve chosen weilu’s implementation here because it’s first result in Google for ‘bip39 javascript’ at the time of writing this.

NOTE: We’ve simplified this implementation here by not using ‘random BIP 32 subpath’ which GreenAddress uses to avoid signing twice using the same key. If you want to use it, you need to pass appropriate random path_hex argument to authenticate() and then use corresponding derived private key.

To derive the private key, you can use the following code:

var mnemonic = '[your mnemonic]';
var cur_net = 'mainnet';  // 'testnet' for testnet
var bip39 = new BIP39();
var seed = bip39.mnemonicToSeed(mnemonic);  // this is slow, perhaps move to a webworker
var key = Bitcoin.HDWallet.fromSeedHex(seed, cur_net);
// NOTE: master priv key shouldn't be used for signing because repeated signing using the
// same key is dangerous, so in production a random BIP32 subpath should be used.

Logging in GreenAddress WAMP server¶

Retrieving a token¶

To connect to GreenAddress server, you need to acquire a token over HTTP which is used for protecting our servers against Denial of Service attacks:

var token = $.get('https://greenaddress.it/token/', init_wamp);  // init_wamp is defined in the next subsection

Connecting via WAMP and using the token¶

Now after retrieving the token, we can finally authenticate using the token:

var init_wamp = function(token) {
        ab.connect('wss://prodwss.greenaddress.it/ws/inv', function(session) {
                session.authreq(token).then(function(challenge) {
                        var signature = session.authsign(challenge, token);
                        session.auth(signature).then(function(permissions) {
                                on_wamp_auth(session);  // defined in the next subsection
                        });
                });
        });
}

Logging in to the wallet and retrieving balance¶

After WAMP authentication, we can use the previously derived private key to log in and retrieve the balance:

var on_wamp_auth = function(session) {
        session.call('http://greenaddressit.com/login/get_challenge',
                        key.getAddress().toString()).then(function(challenge) {
                var challenge_bytes = new Bitcoin.BigInteger(challenge).toByteArrayUnsigned();
                var signature = key.priv.sign(challenge_bytes);
                signature = Bitcoin.ecdsa.parseSig(signature);

                session.call('http://greenaddressit.com/login/authenticate',
                                [signature.r.toString(), signature.s.toString()], false).then(function(data) {
                        // data contains some wallet configuration data
                        session.call('http://greenaddressit.com/txs/get_balance').then(function(balance) {
                                console.log(balance.satoshi);
                        });
                });
        });
};
